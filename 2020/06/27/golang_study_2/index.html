    自增自减

    Go语言中 自增自减 （++ 和 – )

    不能赋值 不能求值

    不能++i 只能i ++

    且 i++是不不能求值的

    因为不是表达式只是语句， 不能x = i++；

    数组和切片

    数组

    数组是值类型

    可以互相赋值，赋值即复制内存，并非指针指向内存(C语言不允许)

    数组得有元素个数 未知是…

    切片不标明元素个数[]

    声明

    var arr [num]type

var arr [...]type

 var arr[5]int

var arr[...]int

    初始化

    var ages = [30]int{1,2,3,4,5,6,7}
var ages2 = [...]int{1,2,3,4,5,6,7}
var ages3 = [...]int{1:10, 2:4, 10:10}

    多维数据的初始化

    var arr1 = [3][2]int{ {1,3}, {2,4}, {5,6} }
var arr2 = [3][2]int{
    [2]int{1,3},
    [2]int{2,4},
    [3]int{5,6},     //这样方式的初始化，最后一个元素也要加逗号
}

    数组遍历

    数组遍历的时候for range 返回索引和值

    a4 := [...]int{1:2,4:2}
    for i,v:=range a4{
        fmt.Println(i,v)
    }

    切片

    切片是引用型，

    类似C的数组

    通过切片可以通过索引修改内存数据，arr[n] = xxx ,此时arr[n]指向的数值已经修改，引用改内存数据的切片的值都发生了变化。

    arr := []int{1,2,3}
arr[0] = 10
此时arr 已经变成[10, 2, 3]

    切片的声明

    类似数组但是[]中括号内没有长度

    var arr = []int{1,2,3}

    通过数组创建切片

    数组： var a1 = [...]int{1,2,3,4,5,6,7,8,9}
切片:  var s1 = a1[:] //此时s1就成为了一个切片， 还可以再切片的基础上再次切片，方法类似python list
            //但是此时s1并不是内存地址而是切片类型
            a1[1:]
            a1[2:5]
            a1[:4]

    make创建切片

    var val = make([]type, len, cap)

    举例：

    var a2 = make([]int, 5, 10) //创建一个长度为5的，容量为10的切片，即开辟一块10个int长度的内存，把地址交给a2，然后对a2解引用， 打印a2 就是解引用之后的切片的值.  如果声明时省略了cap则默认cap和len一样
此时打印a2的值是[ 0 0 0 0 0 ] 即5个0

    make创建的切片即使没有初始化，内存长度为0也不为nil， 而如果只声明了一个切片类型的变量而没有初始化，则该变量等于nil.

    var val []int  // val == nil
var val = make([]int, 0, 0)  // val != nil 
var val = []int{}   // val != nil，此时切片是长度0容量0的地址，非nil

    append

    arr1 := []int{1,3,5}
arr1 = append(arr1, 7)
arr2 := []int{9,11}
arr1 = append(arr1, arr2...) //解引用arr2 
append之后 的cap增长，1024之前是翻倍， 1024之后是1/4，
append之后的内存地址是新申请的， 与c的realloc 不一样。

    copy

    copy(dest, src) 
copy切片 其实是把内存页复制了一份， 也就是复制的是值，不是地址

    切片元素的删除

    切片的元素通过append将指向的值copy到目标地址，然后将地址重新指向到目标地址

     删除了第二个元素即下标是1的值    
a3 := []int{1,2,3,4,5}
a4 := a3
a4 = append(a4[:1], a4[2:]...)   
fmt.Println(a4)
fmt.Println(a3)
---
[1 3 4 5]
[1 3 4 5 5]

类似如下操作： 
copy(a4[1:],a4[2:])  // 把目标元素后边的所有值copy到目标元素开始的位置
a4 = a4[:len(a4)-1]  // 对a4再切片， 从最后切掉要删除的个数1

    指针和内存分配

    Go语言不能进行指针运算，只能查找赋值，但是写法比较反人类

    取地址和解引用 

    var ptr * int = new(int) //go
int * a  = (int *) malloc(sizeof(int)) // c

    make()是给slice,map,chan申请内存，返回值本身（或者说返回值后，使用的时候会被go解引用）

new()给基本上的数据类型申请内存，返回内存地址，类似malloc()，但是会初始化赋值0，在go中很少用到

    Map

    类似python的dict

    var m1 map[string]int // [key type]value type

初始化:
m1 := map[string]int{}
var m1 = map[string]int{}

var m1 = make(map[string]int, 10, 10)

    元素存在map中

    value, ok := m2["alice"]

如果该key不存在则ok的值是false，值value就是0; 
如果存在value就是key的value。

    闭包

    函数内不能定义有名字的函数， 可以定义匿名函数

    func f1(f func()){
  fmt.Println("in f1")
  f()
}

func f2(x,y int){
  fmt.Println("in f2")
  fmt.Println( x + y )
}

func f3(f func(int,int), x, y int)func(){
  tmp := func(){
    f(x, y)
  }
  return tmp
}
func main(){
  ret := f3(f2, 300, 200)
  f1(ret)
}

    注意事项

    Go语言变量不初始化会有默认的值0，类似calloc, 会对变量进行默认0赋值；

    C中如果不初始化，则只申请了内存，值不一定是什么。

    2. 写一个程序，统计一个字符串中每个单词出现的次数。比如:”how do you do “中how=1 do=2 you=1

      package main

import (
    "fmt"
    "strings"
)

func main(){
    var ch1 string = "how do you do"
    s1 := strings.Split(ch1, " ")
    m1 := make(map[string]int, 4)
    for _,v1:=range s1{
        _,ok := m1[v1]
        if !ok {
            m1[v1] = 1
        }else{
            m1[v1] ++
        }
    }
    fmt.Println(m1)
}

    2. 回文判断

      package main

/* 判断字符串是否是回文
** 上海自来水来自海上
*/
import (
    "fmt"
    "strings"
    "os"
)
func main(){
    var s1 = "上海自来水来自海上"
    var sli = strings.Split(s1, "")
    var s1Len = len(sli)
    var midIndex = s1Len/2 + s1Len%2 -1
    var  maxIndex = s1Len - 1 
    for i,v := range sli{
        if i == midIndex{
            break
        }else{
            if v != sli[maxIndex -i ]{
                fmt.Println("不是回文")
                os.Exit(1)
            }
        }
    }
    fmt.Println("是回文")
}
