    Hexo+Github自建博客

    其他 hexo
    发布时间 : 2020-06-25 00:00
    评论:
    1. 概述
    2. 部署步骤

    概述

    本文主要是把自己部署github pages和hexo的过程进行记录，为近期使用此方法部署的同志们提供参考。

    国内由于访问github比较慢，可以用gitee代替github，方法同样适用，但是gitee的由于对我下载的主题支持的不好，所以又改回github。

    环境：

    macOS 系统

    本地部署

    Nodejs,hexo,github

    部署步骤

    1. 下载安装git ， brew install git （brew不会用可以google，或者手动下载安装）

      下载并安装nodejs: nodejs下载，下载完是个pkg，直接安装即可，如图：

      • node.js其官方网站开始https://nodejs.org/zh-cn/
      • Git官方网站开始https://git-scm.com/

    1. 安装npm，通过npm部署hexo

      brew install npm
修改阿里的npm源: npm config set registry https://registry.npm.taobao.org
查看npm源是否修改成功： npm config get

    2. 创建一个本地blog项目的文件夹: myblog_web

    3. 再安装hexo,执行 npm install hexo-cli -g

      Note:

      或者安装完整的Hexo npm install hexo

      Linux的话需要把node_modules目录加入到环境变量

      echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

      这部分来自hexo项目hexo官网的说明

    1. 初始化hexo项目，进到myblog_web目录，执行hexo init或者hexo init path/myblog_web，然后等着自动从git上拉取项目到本地。

      项目拉取完成之后可以通过npm fund 查看项目的依赖信息

      ➜  myblog_web npm fund
hexo-site@0.0.0
├─┬ https://opencollective.com/hexo
│ └── hexo@4.2.1
├─┬ https://github.com/sponsors/ljharb
│ └── resolve@1.17.0
├─┬ https://github.com/chalk/ansi-styles?sponsor=1
│ └── ansi-styles@4.2.1
├─┬ https://github.com/sponsors/jonschlinkert
│ └── picomatch@2.2.2
└─┬ https://github.com/sponsors/isaacs
  └── glob@7.1.6

    2. 生成静态页面，执行hexo g 或者 hexo generate生成网站(每更新一次都要重新生成静态页面)，生成的页面在myblog_web/public文件夹

      ➜  myblog_web hexo g
INFO  Start processing
INFO  Files loaded in 105 ms
INFO  Generated: index.html
...
INFO  Generated: 2020/06/24/hello-world/index.html
INFO  28 files generated in 295 ms

    3. 执行hexo s -p 8080 启动本地服务，即可通过本地http://localhost:8080访问页面.

      Note:

      不指定端口的话就是默认端口4000

      加–debug 可以在屏幕显示调试日志

    4. 下载一个自己喜欢的主题，可以从hexo官网下载: https://hexo.io/themes/ ，调整好主题的配置，比如我选择的是https://yelog.org/的，对应的主题都有相关配置说明。

    5. 在github上 创建一个仓库，仓库名保持和uername一致的username.github.io ，如图

    6. 发布到github需要修改_config.yml, 在最下边deploy处修改内容如下：

      # Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
 type: git
 repo: https://github.com/wanglin0c/wanglin0c.github.io.git #仓库的https地址
 branch: master
# gitpage只能使用master分支

    1. cd到myblog_web路径下，删除旧的静态文件并生成新的静态文件，并发布,然后会要求输入github的用户密码，然后上传完成。

      安装插件：
npm install hexo-deployer-git --save 安装该插件可以使每次push到origin remote的时候用--force的方式强制更新。
发布命令: 
hexo clean && hexo generate && hexo deploy   
# 后两个命令可以简写成 hexo g && hexo d

      ​

    1. 添加博客文章，在source/_posts目录下添加markdown文件，需要注意的是每篇markdown文档的开头需要添加开头部分内容（可选，页面目录生成使用）：

      ---
title: 文档名称
date: 日期
tags: 标签
categories: 类别
---
    1. 使用typora编辑MD文档时添加图片的方法：
      • 配置typora的图像设置部分，偏好设置>图像，选择复制到指定路径，并勾选”优先使用相对路径”，这样在粘贴图片到文档上的时候，就会在文档所在目录创建一个同名的目录存放该文档上的图片，并且图片的路径显示为该文件夹的相对路径，如“blog_hexo/xxxxx.png”，此时typora就可以显示该相对路径的图片，如图：

      • 修改hexo配置，如我的项目myblog_web目录下的配置文件_config.yml, 修改post_asset_folder的值为true，即使用相对路径。

        post_asset_folder: true

      • 本地安装插件， 执行命令npm install hexo-image-link --save 解决typora图片路径问题，本地预览的时候就可以看到图片了。
